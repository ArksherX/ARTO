# Contributing to VerityFlux 2.0

## Getting Started
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Run tests
5. Submit a Pull Request

## Code Style
- Follow PEP 8
- Add docstrings
- Include type hints
- Write unit tests
